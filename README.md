実際には Google は 無限ループリダイレクト（ループ型ドアウェイページ） を自動的に検出し、評価が非常に低くなります。理由を整理します。

⸻

1️⃣ 無限リダイレクトの構造
	•	ページ1 → ページ2
	•	ページ2 → ページ1
	•	（JSリダイレクトでループ）

特徴：
	•	実質コンテンツがない
	•	ユーザーはページを閲覧できない
	•	検索エンジンを欺く意図が明確

⸻

2️⃣ Googleから見た評価

Googleは以下の理由でスパム判定します：
	1.	ユーザーに価値がない
	•	ページを開いてもコンテンツが表示されず、何も得られない
	2.	ドアウェイページ（Doorway Pages）
	•	検索結果のためだけに作られ、最終的に別サイトに誘導する
	3.	不自然な挙動
	•	無限リダイレクトは明らかに「検索エンジンを操作する意図あり」と認識される
	4.	クロール負荷が増加
	•	Googlebotがループで無限にアクセスしてしまうため、リソースの無駄として低評価

⸻

3️⃣ 実際の扱い
	•	インデックスされない（検索結果に表示されない）
	•	Manual Action（手動ペナルティ）の対象になる可能性
	•	サイト全体の評価も大きく下がる

⸻

4️⃣ 比較：正常なリダイレクト

安全な方法：
	•	1ページ → 2ページに 301リダイレクト
	•	ループは絶対に作らない
	•	コンテンツはちゃんと表示する

Googleはこれを自然な移転やURL整理として扱う。

⸻

🔥 結論

JSでページ間を無限ループさせる構造は、ユーザーに価値がない「誘導・欺瞞ページ」と見なされ、 完全にスパム扱い です。
インデックスされず、サイト全体に悪影響が出ます。
